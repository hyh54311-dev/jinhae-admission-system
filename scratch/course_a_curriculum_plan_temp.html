🌟 [청소년 SW동행프로젝트] 책임안전 AI 코스 A 교육과정 안내

🟢 코스 A : 선 넘지 않는 AI 크리에이터 (기본/설계)

인공지능(AI) 시대, 데이터 편향을 극복하고 실생활 문제를 해결하는 AI 창작자 및 개발자로 성장하기 위한 코스 A 교육과정 상세 안내입니다. 본 코스는 코딩 장벽 없이 이미지/텍스트 데이터를 직접 수집 및 훈련시키고 노코드 빌더를 활용하여 세련된 책임안전 AI 웹 서비스(MVP)를 완성하는 것을 목표로 합니다.

📅 1. 전체 16차시 요약 로드맵

📝 2. LMS 워크북 상세 학습 내용 (1~3차시)

학생들이 LMS 디지털 학습지를 통해 수행하고 과제(캡처 및 텍스트)를 제출해야 하는 실제 활동지 구성입니다.

📁 [1차시] 데이터로 만드는 AI (워크북 Page 1~2)

  • 핵심 개념: 인공지능이 데이터를 학습하여 규칙을 스스로 찾는 원리를 깨닫고, 규칙 기반 SW(기존 프로그램)와 머신러닝의 차이점을 구별합니다.
  • 실습 활동 1-1: 규칙 기반 SW vs AI 해결 사례
    • 내용: 실생활 제품의 규칙 기반 소프트웨어가 갖는 문제점과 이를 AI로 해결한 부분 비교 작성
    • 예시: (기존) 로봇청소기가 문턱 높이만 인식하다 현관 밖으로 떨어짐 $
      ightarrow$ (AI) 거실/현관 공간 이미지를 학습하여 밖으로 나가지 않음
  • 실습 활동 1-2: Teachable Machine 이미지 분류 (웹캠)
    • Google Teachable Machine을 활용하여 웹캠으로 이미지를 추가하고, 학습 과정을 거쳐 실시간 추론을 확인합니다.
    • LMS 제출: 이미지 프로젝트 생성, 데이터 수집, 학습 진행, 모델 추론 단계 캡처본 업로드
  • 실습 활동 1-3: Teachable Machine 이미지 분류 (데이터셋 파일)
    • 제공된 이미지 파일셋을 다운로드받아 업로드하여 모델을 훈련하고 검증합니다.
  • 학습 정리 및 피치: 지도학습 모델 생성 3단계 퀴즈 풀기 및 핵심 요약 문장 작성

📁 [2차시] 데이터의 배신 (워크북 Page 3~4)

  • 핵심 개념: 텍스트 분류 인공지능 모델을 구축하고 학습 데이터의 크기나 편향에 따른 성능 저하 현상을 몸소 검증합니다.
  • 실습 활동 2-1: Machine Learning for Kids 텍스트 분류
    • 머신러닝 포 키즈 사이트에 접속하여 텍스트 분류 프로젝트 및 레이블을 생성하고 훈련 데이터를 추가합니다.
  • 실습 활동 2-2: 텍스트 분류 모델 1차/2차 검증
    • 아래 5가지 테스트 문장에 대해 학생들이 '예상 레이블'을 먼저 추정한 뒤, 학습 모델의 추론 결과(1차 검증)와 비교합니다.
      1. "010-1111-2222로 연락해"
      2. "나 부산 살고 있어."
      3. "내 이름은 영희야"
      4. "너 저녁에 뭐 먹을 거야?"
      5. "내 통장 비밀번호 알려줘야 해?"
    • 오판이 발생한 문장을 확인한 후, 추가 데이터를 훈련 셋에 보강하여 모델을 다시 학습(Re-training)시키고 2차 검증 결과와 비교 분석합니다.

📁 [3차시] AI가 넘으면 안되는 선 (워크북 Page 5~6)

  • 핵심 개념: 인공지능 보안 및 디지털 안전의 기초로서, SNS 상에 공유되는 사진 속에서 어떠한 정보가 무심코 유출되는지 분석하며 개인정보 유출에 대한 경각심을 높입니다.
  • 실습 활동 3-1: SNS 게시글 속 개인정보 유출 사례 추적
    • 사례 1 (예시): 공유된 이미지 속 명찰이나 교과서를 통해 학교 이름(동행중학교), 거주지역(서울 강남), 학년(중2)이 특정됨을 확인
    • 사례 2~5 분석: 각각 제시된 사진(칠판 글씨, 창밖 풍경, 택배 운송장 노출 등)을 자세히 관찰하여 숨겨진 개인정보를 표에 추출하여 기록
  • 실습 활동 3-2: 디지털 정보 공개 성찰 토론
    • 성찰 질문: "이러한 개인정보가 SNS에 공개된다면 발생할 수 있는 문제 중 어떤 부분이 가장 우려되나요?"에 대해 자신의 가치관을 서술

💡 3. 지도 교사를 위한 실습 준비 사항 및 팁

  1. 학생 구글 계정 사전 준비: Teachable Machine 및 Machine Learning for Kids 등 모든 클라우드 도구를 원활하게 연동하고 진행하기 위해 학생들의 개인 구글 계정(아이디/비밀번호)을 미리 확인해 두어야 합니다.
  2. 노코드의 교육적 장점 활용: 본 코스는 파이썬 프로그래밍이 들어가지 않으므로, 코딩의 복잡함보다는 '양질의 데이터 입력'이 어떻게 성능 개선을 이끄는지 학생들이 몸소 이해할 수 있도록 2차시 재학습 과정에서 충분한 데이터를 수집하도록 조력해 주십시오.
  3. 윤리적 흐름 연계: 3차시 개인정보 유출 분석 활동은 5~8차시 가드레일 설계(개인정보 입력을 사전 감지하여 필터링하는 챗봇 기획)의 중요한 배경이 되므로, 인공지능이 왜 사람의 안전을 위해 가드레일이 필요한지 윤리적인 관점을 함께 토론하도록 이끌어 주시기 바랍니다.
