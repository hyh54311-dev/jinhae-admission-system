글로벌 올웨더 자산배분 자동화 봇 (Option A) 실전 배포 및 운용 가이드 (GCP 서버리스 버전)

이 문서는 한국투자증권 서브 주식계좌를 추가 개설하여 미국 직상장 ETF(VOO, TLT 등)로 글로벌 올웨더 자산배분을 연간 리밸런싱 및 매월 적립식 투자로 구동하기 위한 단계별 매뉴얼입니다. (구글 클라우드 펑션 서버리스 배포 방식)

[단계 1] 한국투자증권 스마트폰 앱에서 서브 주식계좌 추가 개설 및 통합증거금 설정하기

기존 K-듀얼 모멘텀 봇과의 자산 충돌을 피하기 위해 올웨더 전용 계좌를 1개 더 만듭니다.

  1. 스마트폰에서 한국투자증권 앱을 켭니다.
  2. 메뉴 ➡️ 계좌개설 ➡️ 비대면 계좌개설을 선택합니다.
  3. 종합거래계좌(위탁계좌)를 추가로 개설합니다. (약 1분 소요)
  4. 새로 개설된 계좌번호 8자리(예: 72xxxxxx)와 상품코드 2자리(기본 01 주식/ETF 거래용)를 확인하여 적어둡니다.
  5. 주의: 새로 발급받은 계좌는 동일한 주민번호(ID) 하에 개설되므로, 기존에 발급받으신 OpenAPI AppKey와 AppSecret을 100% 그대로 공유할 수 있어 새로 키를 발급받을 필요가 없습니다.
  6. [★가장 중요★] 통합증거금 서비스 신청하기
  7. 매달 입금되는 원화(KRW)를 봇이 자동으로 감지하여 달러로 즉시 주문할 수 있도록 통합증거금을 켭니다.
  8. 앱 검색창 ➡️ '통합증거금' 검색 ➡️ '해외주식 통합증거금 서비스 신청'을 선택하여 가입합니다.
  9. 신청 대상 계좌를 새로 만든 서브 계좌(72xxxxxx)로 선택한 뒤 신청을 완료합니다. (무료 서비스)

[단계 2] 로컬 .env 파일에 새 계좌번호 등록하기

새 계좌번호를 봇이 인식할 수 있도록 워크스페이스 안의 .env 설정 파일을 수정합니다. (현재 이 단계는 인공지능이 로컬 PC의 파일 수정을 이미 자동으로 마쳐둔 상태이므로, 업로드만 진행하시면 됩니다.)

  • 확인용 .env 내부 설정값: env # 실전 올웨더 전용 서브 주식계좌(01) 번호 8자리 KIS_ALL_WEATHER_CANO=72394127

[단계 3] 구글 클라우드 콘솔 접속 및 클라우드 쉘(Cloud Shell) 열기

서버리스(Cloud Functions) 배포를 위해 구글 브라우저 터미널을 실행합니다.

  1. 웹 브라우저를 열고 구글 클라우드 콘솔에 접속하여 로그인합니다.
  2. 화면 오른쪽 맨 위 헤더 영역의 'Cloud Shell 활성화' 아이콘(돋보기 옆, >_ 모양의 미니 터미널 아이콘)을 클릭합니다.
  3. 브라우저 하단에 검은색 터미널(Cloud Shell) 창이 나타납니다.

[단계 4] 파일 업로드 및 자동 배포 스크립트 실행하기

  1. Cloud Shell 터미널 창의 오른쪽 위의 점 3개 아이콘(더 보기)을 클릭하고 [업로드(Upload)]를 선택합니다.
  2. 내 PC의 프로젝트 폴더(d:\OneDrive - 경상남도교육청\바탕 화면\진해고등학교\2026학년도\antigravity_folder)에서 다음의 3개 파일을 다중 선택하여 업로드합니다:
  3. all_weather_quant_bot.py
  4. gcp_deploy_all_weather.sh
  5. .env
  6. 업로드가 완료되면 Cloud Shell 터미널에 아래 명령어를 입력하여 배포 헬퍼 스크립트를 실행합니다: bash bash gcp_deploy_all_weather.sh
  7. 이 스크립트가 실행되면 내부적으로:
  8. all-weather-bot 이라는 이름의 구글 클라우드 펑션(GCP Cloud Function v2)이 생성 및 배포됩니다.
  9. 매일 밤 11시 00분(23:00 KST)에 월~금요일 평일마다 봇을 깨우는 구글 클라우드 스케줄러(Cloud Scheduler) 크론 잡(all-weather-daily-job)이 자동으로 자동 등록됩니다.
  10. "🎉 Google Cloud Serverless Setup Completed!" 메시지가 출력되면 모든 설치가 무사히 종료된 것입니다.

[단계 5] 일상 운용 및 텔레그램 리포트 확인 방법

등록 완료 후 봇은 매일 밤 11시에 혼자 실행되어 계좌의 자금을 확인하고 운용합니다.

  • 케이스 A: 일반적인 날 (모니터링 모드)
  • 날짜가 연간 리밸런싱 날(1월 2일)이 아니고, 신규 적립금도 들어와 있지 않다면 봇은 아무런 매매를 하지 않고 현재 올웨더 포트폴리오 비중 현황만 조용히 텔레그램 보고서로 발송하고 10초 만에 종료됩니다.
  • 케이스 B: CMA 계좌에 원화(KRW)를 적립한 날 (적립식 투자 모드)
  • 봇이 원화 예수금(cash_krw)과 달러 예수금(cash_usd)을 합산한 총 달러 구매력이 $100 USD를 넘는 것이 감지되면 자동으로 실행됩니다.
  • 기존 자산은 하나도 매도하지 않고(양도세 22% 회피), 오직 그 입금액 한도 내에서 비중이 가장 부족한 자산들만 골라서 매수(Buy-Only)하여 비중을 맞춰줍니다. 매수 완료 시 텔레그램으로 적립 완료 리포트가 발송됩니다.
  • 이때, 한국투자증권의 통합증거금 서비스가 활성화되어 있으므로, 봇이 보낸 달러 주문은 계좌 내 원화 자금으로 자동 정산되어 체결됩니다.
  • 케이스 C: 매년 1월 2일 (정기 리밸런싱 모드)
  • 1년에 한 번 정기일에 도달하면 비중이 초과한 자산을 매도하고 부족한 자산을 매수하여 전체 포트폴리오 비중을 정확히 30 : 40 : 15 : 7.5 : 7.5로 완전 리셋하고 리포트를 보냅니다.
