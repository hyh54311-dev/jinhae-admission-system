# 3학년 화법과작문 수행평가 수기 입력 가이드

본 문서는 수행평가 미응시자 추가 시험 등 학생들의 서술형 답안을 수기로 구글 시트에 직접 입력해야 할 때 사용하는 표준 가이드라인입니다.

---

### 1. 핵심 원칙: 구글 시트 직접 입력 (우회 표준)
* **문제점**: 교육청/학교 Workspace 도메인 제한이 걸린 구글 폼은 로그인하지 않은 익명 HTTP POST(Python requests 등) 요청 시 `200 OK` 응답 코드와 함께 로그인 안내 페이지 HTML을 리턴하므로, 전송 성공으로 오판하나 실제 데이터는 적재되지 않는 문제가 있습니다.
* **표준 해결책**: 구글 폼 우회 제출을 시도하지 않고, **연동된 구글 스프레드시트 API를 활용해 시트에 직접 데이터를 삽입(Append)합니다.** 이는 `token.json` 구글 인증을 활용하여 도메인 제한의 영향을 전혀 받지 않으므로 100% 안전하고 확실합니다.

---

### 2. 구글 스프레드시트 구조 규격
* **스프레드시트 ID**: `1P3EuWeLlxjmozKGPpow279Qyc_moSGt4QAzx68SqJBU` (구글 드라이브 내 '제목 없는 스프레드시트')
* **대상 탭 이름**: `설문지 응답 시트6`
* **열 구성 (17개 열 - 0부터 시작하는 인덱스 기준)**:
  * `[0]`: 타임스탬프 (형식: `YYYY. M. D. 오전/오후 H:MM:SS`)
  * `[1]`: 반 선택 (값 예시: `"8반"`, `"6반"`)
  * `[2] ~ [11]`: 각 반별 이름 선택 (1반은 index 2, 2반은 index 3, ..., 10반은 index 11). **반드시 해당 학생의 학반 열에만 "번호 이름" 형태로 기입하고, 다른 9개 반의 이름 열은 `""`(빈 값) 처리해야 함.**
  * `[12]`: 16. 세런디피티의 개념을 한 문장으로 쓰시오.
  * `[13]`: 17. <보기>의 연구를 수행한 사람의 이름을 쓰시오.
  * `[14]`: 18. 플레이밍이 항박테리아 제제를 개발하려는 동기가 약했던 까닭을 두 가지를 <조건>에 맞게 쓰시오.
  * `[15]`: 19. 페니실린이 박테리아를 억제하는 까닭을 조건에 맞게 서술하시오.
  * `[16]`: 20. 그람 음성균이 페니실린에 효과가 없는 까닭을 <조건>에 맞게 서술하시오.

### 3. 필수 검증 수칙
1. **실시간 적재 확인**: 데이터 추가 API 호출 직후, 반드시 해당 스프레드시트의 값을 다시 조회(Read)하여 **실제 입력 완료 여부 및 행 수 증가를 확인**해야 합니다.
2. **이름 중복 및 전후 비교**: 입력 전 해당 학생이 이전에 제출한 내역(기존 타임스탬프)이 있는지 이름을 우선 검색하고, 중복 제출이 발견되면 기존 데이터와의 정합성을 비교하고 중복을 정리해야 합니다.
3. **로컬 엑셀 백업**: 데이터가 입력된 후에는 로컬 파일인 `3학년_화법과작문_수행평가_양식.xlsx`에도 동일한 행을 백업해 두어야 합니다.
