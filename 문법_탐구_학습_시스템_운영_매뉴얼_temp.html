AI 협업 귀납적 문법 탐구 학습 시스템 운영 매뉴얼

본 매뉴얼은 고등학교 2학년 국어과 교육과정에 맞춘 'AI 협업 귀납적 문법 탐구 학습' 시스템의 설계 개요, 구성 요소, 배포 지침, 그리고 교과 수업 적용 시나리오를 안내합니다.

1. 수업 설계 및 교육과정 연계 개요

📖 성취기준 및 단원 연계

  • 연계 성취기준: 2022 개정 화법과 언어 [12화언01-02] 표준 발음을 이해하고 정확하게 발음하는 국어생활을 한다.
  • 학습 단원: 국어 음운의 변동 (교체, 탈락, 첨가, 축약)

💡 귀납적 탐구 모형 설계 철학

기존의 주입식 문법 수업에서 벗어나, 학생들이 제시된 국어 언어 자료(실제 발음 현상)를 관찰하여 스스로 공통된 규칙성(가설)을 발견하는 귀납적 탐구 모델을 지향합니다.
- AI 튜터와의 Socratic 대화: 학생이 최초로 수립한 규칙(가설)에 오류나 예외가 있을 경우, AI는 정답을 직접 말해주는 대신 반례(Counter-examples)를 제시하여 학생 스스로 생각을 정교화하도록 유도합니다.
- 성장 궤적 기록: 학생들이 최초로 세운 오개념(가설)이 AI와의 대화 과정을 거쳐 어떻게 다듬어졌는지 스프레드시트에 고스란히 기록되며, 이를 기반으로 학교생활기록부 교과세특 초안이 유기적으로 자동 작성됩니다.

2. 시스템 구성 및 파일 아키텍처

본 시스템은 구글 스프레드시트(데이터베이스)와 구글 Apps Script(백엔드 로직), 그리고 HTML/JS(사용자 화면)로 결합되어 동작합니다.

🗂️ 구성 파일 역할 명세

파일명 파일 형식 주요 역할 및 기능
Code_Grammar.gs 구글 Apps Script 웹앱 서빙(doGet), 명렬 데이터 로드, 대시보드 API serving, Gemini 피드백 API 호출 및 세특 배치 생성 엔진
index_grammar.html HTML/JavaScript 학생용 모바일/PC 입력 화면. 주제 선택, 최초 가설 입력(100자 이내 제한), AI 실시간 Socratic 대화 피드백창
dashboard.html HTML/CSS/JS 교사 탁상용 실시간 포스트잇 모니터링 대시보드. 15초 자동 새로고침, 질문 이력 파싱 및 대형 발표 전광판 연동

3. 핵심 시스템 기능 상세

🤖 A. Socratic AI 피드백 엔진

  • 의미 없는 입력 차단 (Nonsense Detection): 'ㅋㅋㅋ', 'asdf', '1111...' 등 문법 탐구와 무관한 의미 없는 문자 나열이 감지될 경우, 기계적 칭찬을 배제하고 "의미 없는 글이 입력되었습니다. 오늘 함께 탐구할 문법 주제와 규칙에 대해 질문이나 가설을 직접 소리 내어 읽어보고 입력해 볼까요?"와 같이 올바른 탐구를 다시 요구합니다.
  • 대화 맥락 인지 (Contextual Interaction): 실시간 피드백 요청 시 웹앱의 누적 대화 기록(chatHistory)이 제미나이 API에 함께 전달됩니다. 학생이 던진 꼬리 질문의 맥락을 완벽하게 파악하여 힌트를 제공합니다.

🎨 B. 실시간 관제 대시보드 (Padlet 스타일)

  • 질문 리스트 노출: 단순 통계용 "대화 글자수" 표시를 완전히 제거하고, 학생이 AI에게 던진 질문들을 "질문 1:", "질문 2:" 와 같이 격자 카드 안에 순차적으로 표시합니다.
  • 가변 높이 포스트잇: 카드 크기를 가로 280px로 유지하되 높이를 가변형(height: auto;)으로 지정하여 질문이 많거나 가설이 길어져도 카드 크기가 늘어나 잘림 없이 전원 노출됩니다.
  • 대형 발표 전광판: 교실 스크린에 띄워둔 대시보드 화면에서 특정 카드를 클릭하면, 화면 아래에 28pt 대형 전광판으로 확대되어 학급 전체 토의용으로 쓰일 수 있습니다.

4. 교사용 설치 및 배포 매뉴얼

1단계: 스프레드시트 사본 만들기

  1. 선생님의 구글 드라이브에 문법 평가용 스프레드시트를 복제하여 사본을 만듭니다.
  2. 스프레드시트 상단 메뉴의 [확장 프로그램] ➡️ [Apps Script]를 클릭하여 코드 에디터 창을 엽니다.

2단계: 파일 구성 및 코드 붙여넣기

  1. 왼쪽 파일 목록의 기존 .gs 파일 내용을 지우고 최신 Code_Grammar.gs 내용을 붙여넣어 저장합니다.
  2. [+ 파일 추가] ➡️ [HTML]을 두 번 눌러 각각 index_grammar 파일과 dashboard 파일을 생성합니다.
  3. 각 HTML 파일에 해당하는 소스 코드(로컬 내 HTML 코드)를 복사하여 붙여넣고 저장합니다.

3단계: 사전 DB 및 API 키 초기화

  1. 구글 스프레드시트 창으로 돌아와 화면을 새로고침하면 상단 툴바에 📝 문법 세특 자동화 메뉴가 생깁니다.
  2. 🔑 Gemini API 키 설정을 눌러 Google AI Studio에서 발급받은 개인 API 키를 입력합니다.
  3. 📖 문법 단어 사전 초기화 메뉴를 클릭하면 표준 발음법 RAG 사전 시트(문법_단어_사전)가 자동으로 생성 및 입력됩니다.

4단계: 웹앱 새 배포 실행 (접속 주소 발급)

  1. Apps Script 창 우측 상단의 [배포] ➡️ [새 배포]를 클릭합니다.
  2. 유형에서 [웹 앱]을 선택하고 다음 설정을 확인합니다.
  3. 웹앱 실행 대상: 나(선생님 구글 계정)
  4. 액세스할 수 있는 사용자: 모든 사용자(Anyone) (로그인 없이 누구나 제출 가능)
  5. [배포]를 완료하고 새로 발급된 웹앱 URL을 복사합니다.
  6. 학생용 화면 접속: 웹앱 URL 그대로 링크 공유
  7. 교사용 대시보드 접속: 웹앱 URL 끝에 ?view=teacher를 붙여서 주소창에 접속

5. 교과 수업 적용 실전 시나리오

🧪 대표 탐구 주제: '교체 (음절의 끝소리 규칙 & 된소리되기)'

탐구 단계 학습 활동 및 학생 입력 예시 시스템 및 AI 반응
1단계: 주제/데이터 관찰 학급과 이름을 선택한 뒤 탐구 주제 교체 (음절의 끝소리 규칙 & 된소리되기)를 선택하고 제시된 언어 데이터(옷[옫], 밖[박], 부엌[부억], 국밥[국빱])를 소리 내어 관찰합니다. 화면에 표준 발음법 관련 예시 표가 노출되어 학생의 귀납적 비교 분석을 돕습니다.
2단계: 최초 가설 작성 관찰을 토대로 최초 규칙을 100자 이내로 입력합니다.
(예시: "음절 끝에 자음이 오면 7개 중 하나로 대표 발음되고, 받침 뒤에 안울림 소리가 연결되면 된소리로 바뀐다.")		 100자 이내로 정상 입력되면 검증 버튼이 활성화됩니다. (글자 수 초과 시 경고 표시)
3단계: AI 힌트 대화 학생이 최초 가설에 대해 [AI에게 가설 검증받기]를 누르면 소크라테스 대화가 시작됩니다.
(학생 질문: "옷[옫]이나 꽃[꼳]은 받침이 ㄷ이 되는데, 신고는 'ㄴ' 받침인데 왜 [신꼬]가 되나요?")		 AI가 질문을 분석하여 정답을 바로 알려주지 않고 힌트를 던집니다.
(AI 힌트: "명사인 '신고'와 동사인 '신발을 신고'의 발음을 비교해 보세요! 단어 성격에 힌트가 있어요.")
4단계: 최종 보고서 제출 AI와의 연속된 대화(최소 5회 필수)를 거쳐 정교화된 최종 가설을 수립하고 제출합니다.
(최종 가설: "음절 끝의 자음은 7개 대표음으로 교체되어 소리 난다. 안울림 받침 뒤의 예사소리는 된소리가 되며, 용언 어간 받침 ㄴ, ㅁ 뒤에 오는 어미 첫소리도 된소리로 발음된다.")		 최종 제출 완료 시 데이터가 스프레드시트에 기입되며 대시보드로 실시간 동기화됩니다.

6. 학생생활기록부 세특 초안 활용 가이드

AI가 생성한 세특 및 평가는 외부 서버로 전송되지 않으며, 선생님이 복제하신 개인 구글 스프레드시트 탭 내부에 안전하게 수집됩니다.

📝 세특 및 결과물 기록 위치 명세

1) 개인별 수행평가 결과 기록 (문법_수행평가_응답 시트)

학생이 개별 탐구 활동을 마친 결과 데이터가 기록되는 공간입니다.
- I열 (세특 초안 (AI)): 학생의 최초 가설 수립 오개념 ➡️ AI 힌트 대화 ➡️ 최종 가설 도출 성장 궤적을 350자 내외로 자동 정리한 생활기록부 세특 초안이 기입됩니다.
- J열 (글자수(바이트)): 공백을 포함한 한글 글자수와 바이트 수(예: 342자 (1026B))가 자동 계산되어 기입됩니다.
- K열 (처리 상태): API 처리가 완료되면 백그라운드 트리거에 의해 대기에서 완료로 변경됩니다.

2) 조별 공동체 결과 기록 (문법_조별_응답 시트)

모둠 단위의 문법 자료 분석 및 통합 보고서 결과 데이터가 기록되는 공간입니다.
- G열 (조별 세특 참고사항 (AI)): 조원들의 협업적 역할 기여도 및 보고서 분석 성과에 기반한 모둠 평가 텍스트가 기입됩니다.
